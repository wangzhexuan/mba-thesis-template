# 第一部分：开始之前

## 1. 三个名词，一次说清

| 名词 | 大白话解释 | 类比 |
|------|-----------|------|
| **Claude Code** | 一个能"看见你电脑里论文文件"的 AI 助手。你跟它说话，它能直接帮你读文件、写文件、改文件 | 一个随叫随到、不睡觉的论文助理 |
| **模板 / 项目文件夹** | 一个装好了流程、表格、范例的文件夹，照着用就行 | 宜家家具的"说明书 + 全套零件" |
| **指令（也叫 Prompt）** | 你发给 AI 的一段话，交代它做什么 | 你给助理派活儿："帮我查 10 篇文献" |

> 💡 你**不需要**记任何 `/命令`。从头到尾，你跟 AI 都是**用大白话**交流。

## 2. 动手前的准备清单

写第一个字之前，先把这几样准备好，能少走很多弯路：

- [ ] **电脑上已经装好 Claude Code**（这是你跟 AI 对话的地方。没装的话，请先按官方说明安装。）
- [ ] **拿到这套工具文件夹**（就是 `mba-thesis-template` 那个文件夹）。
- [ ] **一个知网（CNKI）账号**——学校一般都有，用来下载真实文献。
- [ ] **想清楚你的大致选题**，哪怕只是个方向（比如"某公司某项业务怎么改进"）。
- [ ] **学校的论文格式要求**（字体、字号、字数、查重率红线、参考文献格式）。找不到也没关系，后面可以让 AI 帮你查。

> ⚠️ **AI 不能替你做的事**：它不能替你上知网下载文献、不能替你做真实的问卷和访谈、不能替你拍板。这些是你的活儿。

# 第二部分：把模板变成"你的论文项目"

**核心原则：模板原件别动，每写一篇论文就复制一份新的。** 这样模板能反复用，多篇论文也互不干扰。

### 第 1 步：复制一份模板

把 `mba-thesis-template` 文件夹**复制一份**，把副本重命名成好认的名字，比如 `我的MBA论文`。

> 不会用命令也没关系：在访达（Mac）或文件资源管理器（Windows）里，**右键文件夹 → 复制 → 粘贴 → 重命名** 就行。

### 第 2 步：填好你的"论文档案"（二选一）

模板需要先知道你的论文基本信息（题目、作者、学校、研究类型等）。有两种填法，**挑最省事的**：

**方法 A（最简单）：用网页向导**
双击文件夹里的 **`init.html`**，会打开一个网页表单，像填问卷一样把信息填进去，然后按提示保存。

**方法 B：直接让 AI 帮你建**
用 Claude Code 打开你的 `我的MBA论文` 文件夹，然后把下面这段话发给 AI（把 `{}` 里换成你的信息）：

```
我要开始写 MBA 论文了，帮我做项目初始化。
题目：{你的论文题目}
作者：{你的姓名}
学校：{学校名}
导师：{导师名}
研究类型：案例研究（或：问卷调查）
请帮我把这些信息填进 config 文件夹的配置里，并告诉我还缺哪些必填项。
```

### 第 3 步：确认一下就开工

让 AI 帮你检查一下有没有漏填：

```
请帮我检查 config 文件夹里的配置是否完整，列出所有必填项和我还没填的项。
```

填完这一步，你的论文项目就建好了。下面进入正式写作。

# 第三部分：学会跟 AI 说话（最重要的一节）

**为什么 AI 经常答非所问？** 90% 是因为你给的信息不够。AI 不认识你、不知道你的论文，你说得越具体，它给得越准。

## 好指令的"黄金三件套"

一条好指令，尽量包含这三样：

1. **背景**：我是谁、我的论文是关于什么的。
2. **要做什么**：这次具体要它帮你干哪件事。
3. **要什么样的结果**：多少字、什么格式、什么风格、几条要点。

### 坏例子 vs 好例子

> ❌ **坏**："帮我写文献综述。"
> （AI 不知道你写什么主题、多少字、参考哪些文献，只能瞎编。）

> ✅ **好**："我在写 MBA 论文，研究某公司的客户管理优化。帮我写文献综述里'客户分层'这一部分，大约 1500 字。请基于我接下来贴给你的这几篇文献的摘要来写，**不要自己编造文献**，按主题归纳（不要一个作者一段流水账），最后指出研究空白。"

## 3 个万能句式（不知道怎么说时套用）

- **要它做事**："我在写……（背景）。帮我……（任务），要求……（字数/格式/风格）。"
- **要它改东西**："下面这段是我写的（或 AI 写的），帮我……（改短/改通顺/降低 AI 味/补充论据）：{粘贴内容}"
- **要它出主意**："关于……，给我 2-3 个方案，并说明各自的优缺点，让我来选。"

> 💡 **一个超好用的习惯**：让 AI 把结果**保存成文件**。直接说"**请保存到 drafts 文件夹，文件名叫 chapter2-literature.md**"，它就会帮你存好，不用自己复制粘贴。

# 第四部分：七步写完一篇论文

下面 7 个阶段，**每一节都是同样的格式**，照着做就行：

> 🎯 **这一步在干嘛** → 📦 **你要先准备什么** → 💬 **复制这句话发给 AI** → 👀 **你要把关什么** → ✅ **做完打勾**

每个阶段的详细版，在模板的 `workflows/` 文件夹里（`00-setup.md` 到 `06-defense.md`）。这里是给小白的精简手把手版。

---

## 第 0 关 · 起步（打地基）

🎯 **这一步在干嘛**：填好论文基本信息和学校格式要求，定下研究方向。

📦 **你要先准备什么**：选题方向、学校格式要求。

💬 **发给 AI**：
```
我是 {学校} 的 MBA 学生，论文方向是 {方向}，研究对象是 {某公司/某机构}。
（1）请帮我把这个方向收敛成 1 个明确的研究问题（不超过 3 个子问题）；
（2）帮我列 3-5 个中文关键词和对应的英文关键词，用来检索文献。
```

👀 **你要把关什么**：研究问题是不是你真正想研究的？别让 AI 替你定方向。

✅ **做完打勾**：
- [ ] 论文基本信息已填进 config
- [ ] 研究问题清晰（≤3 个）
- [ ] 关键词列好了

---

## 第 1 关 · 找文献 + 写综述

🎯 **这一步在干嘛**：找到真实文献，分类整理，写出文献综述。

📦 **你要先准备什么**：**这一步你必须亲自上知网**。AI 不能替你登录知网。

> **正确顺序（重要）**：先让 AI 帮你"想检索词" → 你拿去知网搜、下载 PDF → 再把 PDF 交给 AI 读和整理。

**① 让 AI 帮你想检索词：**
```
我的论文研究 {主题}。帮我生成几条适合在知网高级检索用的检索式，
限定核心期刊（CSSCI / 北大核心）、近 5 年。
```

**② 你自己上知网**，用这些检索式搜、筛选、下载 PDF，放进项目的 `sources/` 文件夹。

**③ 把下载的文献交给 AI 整理：**
```
这是我下载的 {N} 篇文献（PDF 在 sources 文件夹）。
请帮我读一下，每篇用 3-5 句话总结核心观点，并按主题帮我分组。
```

**④ 写综述（关键：先喂材料，再让它写）：**
```
基于上面这些文献的摘要，帮我写文献综述的"{某主题}"部分，约 1500 字。
要求：按主题归纳、不要一个作者一段流水账、做批判性比较、最后点出研究空白。
**严禁编造文献**，只能用我提供的这些。
```

👀 **你要把关什么**：⚠️ **逐条核对 AI 引用的文献是不是真的**——AI 可能会"顺手编"几篇不存在的。每一条都要能在知网搜到。

✅ **做完打勾**：
- [ ] 文献都是你亲自下载的真文献
- [ ] 综述按主题写、不是流水账
- [ ] 引用全部核对过、能查到

---

## 第 2 关 · 搭框架（定骨架）

🎯 **这一步在干嘛**：选好用什么理论、定章节结构、画出研究框架图。

💬 **发给 AI**：
```
我的论文研究 {对象} 的 {问题}。候选理论我知道有 {列出你听过的，如 STP、4C、SWOT}。
（1）帮我推荐最适合的理论组合，并说明为什么适合我这个问题；
（2）按标准 MBA 五章结构，帮我列出每一章、每一节的详细提纲。
```

接着可以让它画框架图：
```
请帮我把研究框架画成一张图（或给我一段清晰的文字描述，我可以用 draw.io 画）。
```

👀 **你要把关什么**：理论别贪多堆砌——能解决你问题的才选。框架要能从"问题"一路推到"对策"。

✅ **做完打勾**：
- [ ] 章节提纲定了
- [ ] 理论选定且说得出理由
- [ ] 研究框架图有了

---

## 第 3 关 · 收数据（找证据）

🎯 **这一步在干嘛**：设计问卷/访谈，收集真实数据，做初步分析。

📦 **你要先准备什么**：⚠️ **问卷和访谈必须是真实的**，AI 只帮你设计，不能帮你编数据。

**① 设计问卷：**
```
我要为 {研究主题} 设计一份问卷，用李克特 7 级量表。
请按 {你的理论维度，如 4C 的四个维度} 设计题目，每个维度 4-5 题，并说明每题想测什么。
```

**② 设计访谈提纲：**
```
我要访谈 {对象，如客户经理}，了解 {主题}。帮我设计一份半结构化访谈提纲，
分几个主题，每个主题 3-4 个开放式问题。
```

**③ 你去发问卷、做访谈、收回数据**，整理成 Excel，然后：
```
这是我收回的问卷数据（Excel 在 data 文件夹）。
请帮我做描述性统计和信度分析（Cronbach's α），并指出有意思的发现。
```

👀 **你要把关什么**：数据必须真实可追溯。AI 给的分析结论，要对照原始数据看一眼对不对。

✅ **做完打勾**：
- [ ] 问卷/访谈来自真实调研
- [ ] 数据整理归档了
- [ ] 信度达标（α 一般 >0.7）

---

## 第 4 关 · 写正文（填肉）

🎯 **这一步在干嘛**：一章一章把论文写出来，配好图表和参考文献。

> **建议写作顺序**：第 3 章（现状分析，数据最清楚）→ 第 4 章（对策）→ 第 2 章（文献综述）→ 第 1 章（绪论）→ 第 5 章（结论）。先写有把握的。

💬 **写某一章（以现状分析为例）**：
```
帮我写第 3 章"{公司} 现状分析"，约 8000 字。
公司概况：{基本信息}
外部环境数据：{你掌握的数据}
调研发现：{问卷/访谈的核心结果}
结构：3.1 公司概况 → 3.2 外部环境 → 3.3 内部分析 → 3.4 调研数据分析
要求：数据标注来源、从现象推到问题、结尾总结 3-5 个核心问题。
```

💬 **写摘要（建议全部写完再写）**：
```
帮我写中文摘要（500-800 字），结构：研究背景→目的→方法→发现→结论→价值。
然后再翻译成对应的英文摘要。关键词 3-5 个。
```

👀 **你要把关什么**：每个结论是不是有数据/文献撑着？AI 容易"说大话"，超出证据范围的要砍掉。

✅ **做完打勾**：
- [ ] 各章初稿完成、字数达标
- [ ] 图表编号连续、标了来源
- [ ] 每个结论都能追溯到证据

## 第 5 关 · 改稿（挑毛病 + 降重）

🎯 **这一步在干嘛**：多角度审查、查逻辑漏洞、降低查重率和 AI 味。

**① 让 AI 当"评审老师"挑毛病：**
```
请像一位严格的评审老师，从选题价值、文献、研究设计、数据分析、论证逻辑、
对策建议、写作规范、学术规范这 8 个方面，审查我这篇论文，按 P0（必改）/P1（建议改）/P2（可改）给出问题清单。
```

**② 查逻辑有没有跳步：**
```
帮我检查这部分论证有没有跳步或"结论超出数据支撑"的地方：{粘贴内容}
```

**③ 降 AI 味 / 降重（模板的招牌功能）：**
```
这段查重/AI 率偏高，帮我降一下：{粘贴段落}
要求：保持原意、事实和引用不变，换表达方式和句式，让它更像真人写的。
```

👀 **你要把关什么**：AI 的修改建议**不要全盘照收**。三个角度（评审、逻辑、术语）都指出的问题 = 一定要改；只有一个角度提的 = 自己判断。改完一定再读一遍，确认意思没改歪。

✅ **做完打勾**：
- [ ] 跑过一轮"评审老师"审查
- [ ] 高优先级（P0）问题都改了
- [ ] 查重率降到学校红线以下、读起来不像机器写的

## 第 6 关 · 答辩（临门一脚）

🎯 **这一步在干嘛**：做答辩 PPT、写讲稿、预演老师可能问的问题。

**① 生成 PPT 大纲：**
```
帮我根据论文做一份 20 页左右的答辩 PPT 大纲，
包含：封面、研究背景与问题、方法与数据、核心发现、对策方案、结论、局限与展望。
```

**② 模拟答辩问答：**
```
请扮演答辩委员，针对我的论文提 10 个可能的问题，
覆盖：方法质疑、数据可靠性、对策可行性、"so what（有什么实际价值）"。
每个问题再给我一个回答思路。
```

👀 **你要把关什么**：MBA 答辩看重"实践价值"，多准备"你的方案怎么落地、对别的公司有啥借鉴"。

✅ **做完打勾**：
- [ ] PPT 做好、控制在 20-25 页
- [ ] 练过模拟问答
- [ ] 论文终稿、PPT、附录都齐了

# 第五部分：最常见的 5 个卡点怎么办

**卡点 1：AI 答非所问 / 给的东西太空泛。**
→ 八成是你给的信息太少。回到[第三部分](#第三部分学会跟-ai-说话最重要的一节)，补上"背景 + 具体任务 + 想要的结果"，再把相关材料贴给它。

**卡点 2：怀疑 AI 引用的文献是编的。**
→ 别信，自己查。把每条引用拿到知网/谷歌学术搜一下，搜不到的直接删。**预防办法**：永远先把真文献喂给它，再让它写。

**卡点 3：查重率太高。**
→ 用第 5 关的"降重"指令，一段一段处理。注意保留事实、数据、引用不变，只换表达。降完再自己读一遍。

**卡点 4：导出的 Word 格式乱（字体、表格错位）。**
→ 把你学校的格式要求告诉 AI，让它按 `config/school-format.yaml` 的设置重新生成。表格错位常见于中文标点，让它检查一下。

**卡点 5：被导师说"一股 AI 味"。**
→ 用模板的降 AI 味工具处理（第 5 关 ③）。核心是打破套路：少用"基于 XX 理论""由此可见""首先其次再次"，多写具体的、有你自己判断的句子。

# 第六部分：三条"保命铁律"

这三条比任何技巧都重要，记牢：

1. **🚫 绝不让 AI 编文献和数据。** 所有引用必须是你亲自找到、能查到出处的真东西。被发现编造 = 学术不端，比写得差严重得多。

2. **✋ 最后拍板的永远是你。** AI 给的是初稿和建议，不是定论。每一个观点、每一条结论，你都要读懂、认可，能在答辩时讲清楚——因为答辩时站在台上的是你，不是 AI。

3. **💾 勤存档。** 每写完一章或一个阶段，让 AI 把结果保存到对应文件夹，并跟它说"**做个 checkpoint（存档）**"。避免辛苦写的东西丢了。

# 第七部分：在不同 AI 工具里使用

> *(Claude Code / Codex / 通用对话 AI)*
>
> 本工具**不是 Claude 专属**。这些资产任何能干活的 AI 都能用：七步工作流、所有 Prompt、文档模板、配置。只有"自动调用 skill"和"要联网/读文件/出文件的 skill"依赖具体环境。

## skill 分两类（决定能不能跨工具）

- **纯指令型（可移植）**：只是"教 AI 怎么写/怎么审"的提示词——写作、文献综述、审稿、逻辑检查、**降 AI 味（humanizer-zh-academic）**、答辩模拟。把对应 `SKILL.md` 喂给任何 AI 都能用。
- **工具型（依赖环境）**：要调脚本/联网/文件工具——查文献、读 PDF、画图、出 Word/PPT。取决于该工具有没有等价能力。

> 模板自带的 skill 只有 1 个：`skills/humanizer-zh-academic/`（纯指令型，跨工具通用）。

## 用 Claude Code（原生最省事）

装好 Claude Code → 打开项目 → 用大白话照着各步 Prompt 说，它会**自动选用**合适的 skill。入口文件 `CLAUDE.md` + `AGENTS.md` 自动生效。本指南前六部分就是按 Claude 的用法写的。

## 用 Codex

1. 在项目目录启动 Codex，先说"**请阅读并遵守 `AGENTS.md`**"（Codex 原生读这个文件）。
2. 按第四部分的七步 Prompt 一步步来，**直接说人话**。
3. Codex 不会自动触发命名 skill，工具型环节改为人工或脚本：

| 原本靠的 skill | 在 Codex 里这样做 |
|---|---|
| 查文献 | 自己上知网 / Scholar 搜，把题录、摘要贴给它整理 |
| 读 PDF | 自己把 PDF 转成文本再贴给它 |
| 降 AI 味 | 把 `skills/humanizer-zh-academic/SKILL.md` 贴给它，说"按这份降 AI 味" |
| 画图 | 让它写 matplotlib 脚本，你本地 `python` 跑 |
| 出 Word | 用模板自带的 `templates/build_thesis.py`，填好内容后 `python` 运行（本就工具无关） |

## 用通用对话 AI（ChatGPT 网页 / Gemini 等）

即使 AI 看不到你的文件也能用：把 `workflows/*.md` 里的 Prompt 连同你的材料一起粘贴进对话；工具型环节同上"人工 + 脚本"。

## 一张表看清每步在哪个工具怎么实现

| 环节 | Claude Code | Codex | 通用对话 AI |
|---|:---:|:---:|:---:|
| 自动选 skill | ✅ 原生 | ⚠️ 手动喂 SKILL.md | ❌ 手动 |
| 七步 Prompt / 模板 / 配置 | ✅ | ✅ | ✅ |
| 降 AI 味、写作、审稿（纯指令） | ✅ | ✅ | ✅ |

> 一句话：**流程 + Prompt + 模板 + 配置全工具通用**；只有"自动触发"和"工具型 skill"在 Claude Code 上最省事，其他工具手动补这部分即可。

# 附录：一页速查表（建议贴墙上）

| 关卡 | 一句话对 AI 说 |
|------|---------------|
| **0 起步** | "帮我把研究方向收敛成 1 个明确的研究问题，并列出中英文关键词。" |
| **1 找文献** | "帮我生成知网检索式" → 自己下载 → "帮我读这些 PDF 并按主题总结" → "基于这些文献写综述，不许编造"。 |
