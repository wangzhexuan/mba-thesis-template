> 本工具提供案例研究型MBA论文的通用结构和写作指引。使用时将 `{占位符}` 替换为实际内容。

## 二、六章制结构（推荐）

```
第1章 导论
├── 1.1 研究背景及研究意义
│   ├── 1.1.1 研究背景（宏观→行业→微观）
│   ├── 1.1.2 研究意义（实践价值 + 应用参考）
│   └── 1.1.3 研究目的
├── 1.2 研究内容与研究方法
│   ├── 1.2.1 研究内容
│   ├── 1.2.2 研究方法
│   └── 1.2.3 技术路线
└── 1.3 研究边界与资料来源

第2章 相关理论与文献综述
├── 2.1 核心概念界定
├── 2.2 理论基础（选择2-3个适配理论）
├── 2.3 国内外研究现状
└── 2.4 文献评述（贡献→不足→切入点）

第3章 {研究对象}现状分析
├── 3.1 企业/组织概况
├── 3.2 外部环境分析（PEST/行业对标）
├── 3.3 内部环境分析（资源能力/SWOT）
└── 3.4 调研数据分析（问卷+访谈+运营数据）

第4章 优化方案设计
├── 4.1 优化目标与原则
├── 4.2 策略一：{名称}（对应问题1）
├── 4.3 策略二：{名称}（对应问题2）
├── 4.4 策略三：{名称}（对应问题3）
└── 4.5 方案小结

第5章 实施保障与效果评估
├── 5.1 组织保障
├── 5.2 制度保障
├── 5.3 资源保障
├── 5.4 风险控制
└── 5.5 效果评估（试点数据/情景预测）

第6章 结论与展望
├── 6.1 研究结论
├── 6.2 研究启示
└── 6.3 研究局限与展望
```

## 三、研究方法组合

案例研究推荐使用"三角验证"：

| 方法 | 作用 | 数据来源 |
|------|------|---------|
| 案例研究 | 深度剖析企业现状 | 内部资料、实地调研 |
| 问卷调查 | 量化员工/客户感知 | 李克特5点量表 |
| 深度访谈 | 管理者视角诊断 | 半结构化访谈 |
| 运营数据分析 | 客观业务指标 | ERP/报表系统 |

## 四、理论框架选择指引

根据论文类型选择适配的理论工具：

| 论文类型 | 推荐理论 |
|---------|---------|
| 营销策略 | STP + 4P/4C/7P + CRM |
| 战略管理 | PEST + 波特五力（慎用） + SWOT + RBV |
| 绩效管理 | BSC + KPI + PDCA + 战略地图 |
| 运营优化 | 精益管理 + 六西格玛 + 价值链 |
| 人力资源 | 胜任力模型 + 激励理论 + 组织行为学 |
| 客户管理 | CRM + 客户生命周期 + 客户旅程 |

原则：只选能解释问题、指导方案的理论，不堆砌模型。

## 五、策略可执行性检查清单

每项策略必须通过以下7要素检查：

| 要素 | 检查问题 | 示例填写 |
|------|---------|---------|
| 对应问题 | 对应第3章哪个问题 | 问题1：客户分层不精准 |
| 责任主体 | 谁负责实施 | 相关业务部门 + 一线负责人 |
| 权限边界 | 是否在现有权限内 | 需总行审批新产品上线 |
| 资源条件 | 需要什么资源 | 预算50万 + 培训2期 |
| 时间安排 | 短/中/长期 | 短期3个月试点 → 6个月推广 |
| 评估指标 | 如何衡量效果 | AUM增长率、客户留存率 |
| 风险控制 | 可能的风险及应对 | 合规风险：加强适当性管理 |
