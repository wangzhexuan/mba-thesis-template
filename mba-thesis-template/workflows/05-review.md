# Phase 5: MBA应用型论文质量审校

## 目标

对MBA案例研究/应用型论文进行质量审校。评审重点不是寻找期刊论文式的理论创新，而是判断论文能否用成熟管理理论解释真实管理问题，并提出可执行、可验证、符合组织约束的解决方案。

## 💡 使用方式

> **你只需要用自然语言告诉AI要做什么，AI会自动调用合适的Skill。**
>
> 本工作流中每个步骤都提供了可直接复制的Prompt示例。
> 你不需要手动输入任何 `/skill` 命令。

## 前置条件

- [ ] 已完成论文初稿（Phase 4）
- [ ] 各章节内容完整
- [ ] 问卷、访谈、内部数据或案例材料已整理
- [ ] 参考文献已整理

## 核心步骤

### Step 1: 应用型MBA综合评审

#### 1.1 评审维度

| 维度 | 评审内容 | 权重 |
|------|---------|------|
| 选题与案例价值 | 问题是否来自真实管理场景，是否有实践价值 | 15% |
| 理论适配性 | 理论和模型是否服务于案例诊断，是否避免机械堆砌 | 15% |
| 案例材料与数据支撑 | 内部数据、问卷、访谈、竞品对标是否可靠充分 | 15% |
| 问题诊断深度 | 是否从现象进入成因，是否识别组织、流程、资源、客户等根因 | 15% |
| 论证闭环 | 背景、现状、问题、成因、方案、保障之间是否前后对应 | 15% |
| 策略可执行性 | 措施是否具体，是否明确责任主体、权限边界、资源条件和时间安排 | 10% |
| 写作与格式规范 | 语言是否清晰，目录、标题、图表、附录是否规范 | 10% |
| 引用与学术诚信 | 文献和数据来源是否可追溯，无编造、无过度引用 | 5% |

> 💬 **Prompt**（综合审校）：
> ```
> 请对我的MBA论文进行应用型质量审校。
> 
> 论文类型：案例研究型 / 问卷调查型
> 论文定位：初稿 / 终稿
> 研究对象：{企业}
> 
> 请按以下8个维度评分（每项/10分），并按P0/P1/P2分级输出问题：
> 1. 选题与案例价值
> 2. 理论适配性
> 3. 案例材料与数据支撑
> 4. 问题诊断深度
> 5. 论证闭环
> 6. 策略可执行性
> 7. 写作与格式规范
> 8. 引用与学术诚信
> 
> 论文内容：
> {粘贴论文内容或提供文件路径}
> ```

#### 1.2 评审流程

1. 通读全文，确认论文定位
2. 抽取章节结构，检查逻辑
3. 对照研究问题，检查每章功能
4. 检查术语、数据口径一致性
5. 标注P0/P1/P2问题

### Step 2: 结构与逻辑闭环检查

#### 2.1 章节功能检查

| 章节 | 应承担的功能 | 常见问题 |
|------|-------------|----------|
| 第1章 | 提出真实管理问题，说明研究价值和方法 | 背景泛化，问题不聚焦 |
| 第2章 | 解释将使用哪些理论工具，以及为什么适配 | 文献罗列，理论和后文脱节 |
| 第3章 | 用案例材料、数据和访谈诊断现状和成因 | 只有现象描述，缺少根因 |
| 第4章 | 根据诊断结果设计优化方案 | 策略和问题不对应，措施过泛 |
| 第5章 | 说明实施路径、资源保障、风险控制和效果评估 | 保障空泛，缺少责任主体和指标 |
| 第6章 | 总结研究结论、管理启示、局限和展望 | 结论重复摘要，局限过于模板化 |

> 💬 **Prompt**（逻辑链检查）：
> ```
> 请检查论文的逻辑链闭合情况：
> 
> 第1章问题：{问题}
> 第2章理论：{理论}
> 第3章诊断：{诊断}
> 第4章策略：{策略}
> 第5章保障：{保障}
> 第6章结论：{结论}
> 
> 请逐项检查：
> 1. 每个策略能否回溯到第3章的问题？
> 2. 每个问题是否有数据/访谈支撑？
> 3. 每个效果目标是否区分了"管理目标"和"因果结论"？
> 4. 涉及权限的建议是否明确了权限归属？
> 
> 输出格式：逻辑链关系图 + 问题清单
> ```

### Step 4: 策略可执行性检查

每项策略必须检查以下7要素：

| 要素 | 检查问题 |
|------|---------|
| 对应问题 | 对应第3章哪个问题或成因 |
| 责任主体 | 谁负责实施 |
| 权限边界 | 是否属于现有权限 |
| 资源条件 | 需要预算、人员、系统还是培训 |
| 时间安排 | 短期、中期、长期如何推进 |
| 评估指标 | 用哪些指标验证效果 |
| 风险控制 | 是否涉及合规、数据安全或声誉风险 |

> 💬 **Prompt**（策略可执行性检查）：
> ```
> 请检查以下策略的可执行性（7要素）：
> 
> 策略一：{策略描述}
> 策略二：{策略描述}
> 策略三：{策略描述}
> 
> 请逐项检查每个策略的：
> 1. 对应问题（第3章哪个问题）
> 2. 责任主体（谁负责）
> 3. 权限边界（是否在权限内）
> 4. 资源条件（需要什么）
> 5. 时间安排（短/中/长期）
> 6. 评估指标（如何衡量）

### Step 6: 降重与降AI率

> **降重**（降低查重率）和**降AI率**（降低AIGC检测率）是两个不同的问题，需要分别处理。

#### 6.1 降重：降低查重率（知网/万方/维普）

查重系统检测的是**文字重复**——你的文字跟已有文献或网络资源的相似度。

> 💬 **Prompt**（逐段降重）：
> ```
> 以下段落查重率较高，请帮我降重：
> {粘贴高重复段落}
> 
> 要求：
> 1. 保持原意和学术性
> 2. 用不同表达方式重写（换词+换句式+换语序）
> 3. 保持事实、数据和引用不变
> 4. 正确使用引用标注（该标引用的标引用）
> 5. 目标：重复率降至15%以下
> ```

> 💬 **Prompt**（全文降重策略）：
> ```
> 我的论文查重率为{比例}%，目标是降到15%以下。
> 请帮我分析哪些部分重复率最高，并制定降重优先级。
> 重点检查：
> 1. 文献综述（容易大面积重复）
> 2. 理论介绍（容易与教材重复）
> 3. 研究方法描述（容易与同类论文重复）
> 4. 数据分析（如果引用了他人数据）
> ```

**降重技巧**：
- 理解后用自己的话表达，不要只换同义词
- 引用部分正确标注，引用不算重复
- 增加原创分析和自己的观点
- 改变句子结构（主动变被动、长句拆短句）

#### 6.2 降AI率：降低AIGC检测率

AIGC检测器识别的不是"内容"，而是**写作模式的统计规律**。AI写作高度可预测，人类写作则更随机、更情境化。

> 💬 **Prompt**（降AI率 — 使用humanizer-zh-academic）：
> ```
> 请帮我降低以下段落的AIGC检测率。
> 这是MBA毕业论文的一部分，属于案例研究型/问卷调查型。
> 
> {粘贴需要处理的段落}
> 
> 要求：
> 1. 先输出风险识别报告（列出命中的AI模式）
> 2. 等我确认后再输出改写版本
> 3. 保持学术语域，不改变论证逻辑
> 4. 注入研究者视角（笔者认为、出乎意料的是等）
> ```

> 💬 **Prompt**（直接改写 — 跳过报告）：
> ```
> 请直接改写以下段落，降低AI率：
> {粘贴段落}
> 
> 文体：MBA毕业论文
> 改写要求：
> 1. 打破"首先/其次/再次"的整齐并列
> 2. 理论名称从段首移到段中
> 3. 删除段末"此案例印证了""由此可见"类总结句
> 4. 替换AI高频词（深入探讨→分析，深刻揭示了→说明了）
> 5. 每段保留2-3处轻微AI特征作为自然噪声
> ```

**16种AI写作模式速查**（改写时对照）：

| 层面 | 模式 | 触发词/特征 |
|------|------|-----------|
| 内容 | 1.理论起笔 | "依据XX理论""基于XX框架"开头 |
| 内容 | 2.套路结尾 | "此案例印证了""由此可见" |
| 内容 | 3.整齐编号 | "首先/其次/再次"等长并列 |
| 内容 | 4.被动套话 | "该处理体现了""该设计基于" |
| 内容 | 5.模板问题 | "面临的核心问题是" |
| 内容 | 6.对称并列 | 三项完全等长的并列句 |
| 内容 | 7.蛇足总结 | "综上所述""不难发现" |
| 内容 | 8.模糊归因 | "专家认为""研究表明"无出处 |
| 内容 | 9.填充短语 | "值得注意的是""需要指出的是" |
| 内容 | 10.泛化结尾 | "具有重要意义""前景广阔" |
| 语言 | 11.AI高频词 | 深入探讨、不可或缺、深刻揭示 |
| 语言 | 12.系动词回避 | "扮演着…角色""充当着…功能" |
| 语言 | 13.过度排比 | 突破了…填补了…创新了…丰富了 |
| 语言 | 14.三维等重 | "从经济/社会/文化三个维度" |
| 风格 | 15.破折号滥用 | 一段内4次以上 |
| 风格 | 16.加粗滥用 | 全文超过5处 |

**7项硬约束（命中即修复）**：

| 约束项 | 上限 |
|--------|------|
| AI高频词 | 每段≤2个 |
| 段末总结套句 | 全文≤1处 |
| 整齐三元并列 | 每段≤1处 |
| "依据XX理论"起笔 | ≤20%段落 |
| 正文加粗 | 全文≤5处 |
| 泛化结尾 | 全文0处 |
| 模糊归因 | 全文0处 |

#### 6.3 降重 vs 降AI率 对比

| 维度 | 降重（查重） | 降AI率（AIGC检测） |
|------|------------|-------------------|
| 检测对象 | 文字与已有文献的相似度 | 写作模式的统计规律 |
| 改写策略 | 换词+换句式+换语序 | 打破模式规律性+注入随机性 |
| 重点区域 | 文献综述、理论介绍 | 全文（尤其分析和结论段） |
| 工具辅助 | humanizer-zh-academic | humanizer-zh-academic |
| 目标 | 重复率<15% | AIGC率<15% |

#### 6.4 综合处理流程

```
推荐顺序：
1. 先完成全文内容（Phase 4）
2. 先做降重（降低查重率） — 改动较大
3. 再做降AI率（降低AIGC率） — 在降重后的基础上微调
4. 最后做格式检查 — 确保改完后格式没乱
```

> 💬 **Prompt**（综合处理）：
> ```
> 请帮我处理以下段落，同时降低查重率和AI率：
> {粘贴段落}
> 
> 处理顺序：
> 1. 先降重：换词换句式，保持原意
> 2. 再降AI率：打破模式规律，注入研究者视角
> 3. 最后检查：学术规范、逻辑连贯、格式正确
> ```
> 请帮我优化以下段落的表达：
> {粘贴段落}
> 
> 要求：
> 1. 用管理语言解释问题，避免空泛理论话术
> 2. 用短句表达关键判断
> 3. 对策略建议写清动作、对象、条件和指标
> 4. 保持学术规范但可读性强
> ```

> 💬 **Prompt**（降重处理）：
> ```
> 以下段落查重率较高，请帮我降重：
> {粘贴高重复段落}
> 
> 要求：
> 1. 保持原意和学术性
> 2. 用不同表达方式重写
> 3. 保持事实、数据和引用不变
> 4. 目标：重复率降至15%以下
> ```
