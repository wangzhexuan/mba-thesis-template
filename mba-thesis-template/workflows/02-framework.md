# Phase 2: 框架设计

## 目标

完成论文的应用分析框架设计和研究方法选择，包括：
- 确定适配案例问题的理论工具
- 设计分析框架
- 选择研究方法
- 制定技术路线
- 绘制研究框架图

## 💡 使用方式

> **你只需要用自然语言告诉AI要做什么，AI会自动调用合适的Skill。**
>
> 本工作流中每个步骤都提供了可直接复制的Prompt示例。
> 你不需要手动输入任何 `/skill` 命令。

## 前置条件

- [ ] 已完成文献综述（Phase 1）
- [ ] 已确定研究问题
- [ ] 已配置 `research-framework.yaml`

## 核心步骤

### Step 1: 确定理论框架

#### 1.1 理论框架选择原则

| 原则 | 说明 |
|------|------|
| 适用性 | 与研究问题匹配 |
| 成熟性 | 理论体系完整 |
| 可操作性 | 能指导实际分析 |
| 情境适配性 | 能解释案例组织的真实约束和管理场景 |

#### 1.2 常用理论框架

**营销管理类**：STP、4P、4C、7P、CRM
**战略管理类**：波特五力（慎用）、PEST、SWOT、价值链、RBV
**运营管理类**：BSC、KPI、PDCA、精益管理、六西格玛

> 💬 **Prompt**（理论选择）：
> ```
> 我的MBA论文研究方向是{方向}，研究对象是{企业}，核心问题是{问题}。
> 请帮我从以下候选理论中选择最适配的2-3个，并说明理由：
> 候选理论：{列出理论}
> 
> 要求：
> 1. 说明每个理论的适用场景
> 2. 分析与本文研究问题的匹配度
> 3. 说明在论文中如何应用（在哪一章用、用在什么地方）
> ```

#### 1.3 配置理论框架

> 💬 **Prompt**（更新配置）：
> ```
> 请帮我更新 config/research-framework.yaml，配置以下理论框架：
> 理论1：{名称} - {应用场景}
> 理论2：{名称} - {应用场景}
> 理论3：{名称} - {应用场景}
> ```

### Step 2: 设计应用分析框架

**案例研究型（case_study）**：
```
分析框架选择：
├── 诊断框架：用于识别现状、问题和成因（PEST/SWOT/价值链）
├── 策略框架：用于把问题转化为优化方案（STP/4P/4C/BSC）
└── 实施框架：用于说明资源、权限、风险和评估
```

**问卷调查型（survey）**：
```
分析框架选择：
├── 理论模型：选择已有成熟量表的理论（TAM/TPB/SERVQUAL/CSI等）
├── 假设推导：从理论和文献推导自变量→因变量→中介/调节关系
└── 研究模型：绘制变量关系图，明确假设路径
```

> 💬 **Prompt**（案例研究型框架）：
> ```
> 请帮我设计MBA论文的应用分析框架。
> 研究对象：{企业}
> 核心问题：{问题}
> 理论工具：{理论1}、{理论2}
> 
> 要求设计三层框架：
> 1. 诊断框架：用什么工具分析现状和问题
> 2. 策略框架：用什么工具设计优化方案
> 3. 实施框架：如何保障方案落地
> 
> 请以层次图的形式展示。
> ```

> 💬 **Prompt**（问卷调查型框架）：
> ```
> 请帮我设计问卷调查型MBA论文的研究模型。
> 自变量：{X1}、{X2}
> 因变量：{Y}
> 中介变量：{M}（如有）
> 调节变量：{W}（如有）
> 理论基础：{理论名}
> 
> 要求：
> 1. 推导研究假设（H1、H2...）
> 2. 绘制研究模型图
> 3. 说明每个假设的理论逻辑
> ```

### Step 3: 选择研究方法

| 研究类型 | 适用场景 | 数据收集 | 数据分析 |
|---------|---------|---------|---------|
| 案例研究 | 深入分析单一/多个案例 | 访谈、文档、观察 | 内容分析、主题分析 |
| 问卷调查 | 大样本量化研究 | 问卷 | 统计分析 |
| 行动研究 | 解决实际问题 | 实践记录 | 反思总结 |
| 混合研究 | 多角度验证 | 多种方法 | 综合分析 |

> 💬 **Prompt**：
> ```
> 我的研究目标是{目标}，研究对象是{企业}。
> 请帮我分析应该选择哪种研究方法，并说明理由。
> 考虑因素：数据可得性、研究深度、样本量、时间限制。
> ```

### Step 4: 制定技术路线

> 💬 **Prompt**：
> ```
> 请帮我绘制技术路线图，包含以下层次：
> 1. 研究起点：{问题}
> 2. 研究过程：
>    - 文献综述与理论选择
>    - 研究设计与方法
>    - 数据收集（{方法}）
>    - 数据分析与问题诊断
>    - 策略设计与优化
>    - 实施保障与效果评估
> 3. 研究终点：结论与展望
> 
> 要求：用流程图形式展示，标注每个阶段使用的方法和产出物。
> ```

### Step 5: 完成第一章写作

> 💬 **Prompt**（撰写第1章）：
> ```
> 请帮我撰写MBA论文第1章绪论。
> 
> 论文题目：{题目}
> 研究对象：{企业}
> 研究背景：
> - 宏观：{政策/行业趋势}
> - 行业：{市场/竞争}
> - 微观：{企业现状/痛点}
> 研究问题：{核心问题}
> 研究方法：{方法}
> 
> 要求：
> 1. 背景从宏观到微观层层递进
> 2. 意义突出实践价值（不强行拔高理论创新）
> 3. 字数约4000字
> 4. 结构：1.1研究背景及意义 → 1.2研究内容与方法 → 1.3研究边界
> ```

#### 第一章结构

```
第1章 绪论
├── 1.1 研究背景及研究意义
│   ├── 1.1.1 研究背景（宏观→行业→微观）
│   ├── 1.1.2 研究意义（实践价值 + 应用参考）
│   └── 1.1.3 研究目的
├── 1.2 研究内容与研究方法
│   ├── 1.2.1 研究内容
│   ├── 1.2.2 研究方法
│   └── 1.2.3 技术路线
└── 1.3 研究边界与资料来源
```

> 💬 **Prompt**（保存文件）：
> ```
> 请将第1章保存到 drafts/chapter1-introduction.md
> ```

## 常见问题

### Q1: 理论框架不知道怎么选？

> 💬 **Prompt**：
> ```
> 我不确定应该用哪些理论框架。
> 我的研究是关于{主题}，研究对象是{企业}。
> 请帮我分析3种可能的理论组合，并推荐最佳方案。
> ```

### Q2: 研究框架图怎么画？

> 💬 **Prompt**：
> ```
> 请帮我生成研究框架图的文字描述，我可以用draw.io绘制。
> 框架包含以下层次：
> 1. 研究问题
> 2. 理论基础
> 3. 分析维度
> 4. 策略设计
> 5. 实施保障
> ```

### Q3: MBA论文一定要写理论创新吗？

不必。重点说明如何把成熟理论用于解决具体管理问题。可改写为"实践价值""应用价值"或"案例启示"。
